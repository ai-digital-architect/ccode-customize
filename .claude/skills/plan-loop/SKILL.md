:mag: Run planning iteration: $ARGUMENTS

You are operating in **planning mode**. You must NOT modify any source code.
Your job is to study the codebase and specifications, then produce a prioritized plan.

## :clipboard: Planning Protocol

### Step 1 — :books: Study Specifications
- Read all files in `specs/` directory
- Build a mental model of what the project should look like when complete
- Note every feature, endpoint, data model, and behavior described in specs

### Step 2 — :mag: Audit Codebase
- Read the current codebase systematically
- Compare implemented features against spec requirements
- Search for:
  - :x: Missing functionality (described in specs but not implemented)
  - :warning: Placeholder or stub implementations (`TODO`, `FIXME`, `throw new Error('not implemented')`)
  - :bug: Known bugs or failing tests
  - :construction: Partial implementations that need completion
  - :lock: Security gaps (missing auth, unvalidated input)

### Step 3 — :bar_chart: Produce Plan
Write `fix_plan.md` with this structure:

```markdown
# Fix Plan
_Last updated: <ISO-8601 timestamp>_
_Generated by: plan-loop iteration_

## :red_circle: Critical (Blocking)
- [ ] <item> — affected files: `<list>` — related spec: `specs/<file>.md`
- [ ] <item>

## :orange_circle: High Priority
- [ ] <item> — affected files: `<list>` — related spec: `specs/<file>.md`

## :green_circle: Medium Priority
- [ ] <item> — affected files: `<list>` — related spec: `specs/<file>.md`

## :white_check_mark: Completed
- [x] <completed items from previous iterations>
```

### Step 4 — :memo: Report
Present:
- Total items found
- Breakdown by priority
- Estimated iteration count (items × ~1.5 iterations per item)
- Recommended focus order

## :warning: Constraints

- **Read-only mode** — do NOT create or modify source files
- **Do NOT modify AGENTS.md** — only the build loop updates AGENTS.md
- **Be exhaustive** — check every spec file against the codebase
- **Be specific** — each plan item must reference concrete file paths and spec sections
